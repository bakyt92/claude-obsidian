# Source: Data Science for Beginners

**Maintainer**: [[Microsoft]] (Azure Cloud Advocates)
**Format**: open GitHub curriculum · 20 lessons / 10 weeks · MIT License · ~35.8k★
**Tech**: Python (Pandas, Matplotlib), SQL + NoSQL, Jupyter notebooks, Azure ML Studio, GitHub Codespaces

> [!key-insight] A beginner on-ramp to [[Data Science]]
> Microsoft's project-based curriculum teaching [[Data Science]] from zero: defining the field and its ethics, working with data (SQL/NoSQL/Pandas), visualization, the data-science lifecycle, and cloud deployment on Azure. Each lesson is quiz-bracketed (pre/post), with knowledge checks, challenges, and assignments. Translated into 50+ languages via GitHub Actions — open education at scale.

## Curriculum arc (6 sections)

| Section | Lessons | Focus |
|---------|---------|-------|
| Introduction | 1–4 | defining data science; **ethics**; defining data; statistics & probability |
| Working With Data | 5–8 | relational (SQL), NoSQL, **Python/Pandas**, data preparation |
| Data Visualization | 9–13 | quantities, distributions, proportions, relationships, *meaningful* viz (Matplotlib) |
| Lifecycle | 14–16 | acquisition → **analyzing** → communication |
| Cloud | 17–19 | cloud intro, low-code training, **Azure ML** deployment |
| In the Wild | 20 | real-world applied projects |

## Why it matters to this vault

- **The beginner counterpart to [[Data Science from Scratch]].** Same domain ([[Data Science]]), opposite teaching strategy: DSFS implements algorithms by hand ([[From-Scratch Pedagogy]]); this curriculum is deliberately **library-first** (Pandas, Matplotlib) and even **low-code/cloud** (Azure ML). The contrast is the lesson: build-from-scratch for understanding vs. use-the-tools for productivity.
- Adds non-modeling parts of the field the books skip: **ethics**, the **data-science lifecycle**, databases (SQL/NoSQL), and cloud deployment.
- Establishes the [[Microsoft]] "For Beginners" open-curriculum pattern as a vault entity (sibling of ML/AI/GenAI for Beginners).

> [!note] Pedagogy contrast, not contradiction
> [[From-Scratch Pedagogy]] says "build it to understand it"; this curriculum says "use libraries to ship." Both are valid for different learners and goals — see the counterpoint noted on [[From-Scratch Pedagogy]].
